A proposito... con il "metodo dell'include", ossia senza compilare la  
libreria ma includendo i singoli files all'interno del programma,  
purtroppo non riesco ancora a far partir niente. E' da ieri che  
faccio tentativi sia sul mac (con GCC 4), sia su Windows (con Visual C 
++), ma quello che ottengo durante la fase di compilazione sono  
sempre errori relativi al linker.

Ad esempio, un codice pur stupido come questo:

#include <iostream>
#include "random.h"
// #include un_po'_di_altri_files_punto_h

using namespace nnfw;
using namespace std;

int main () {
	Random::setSeed(0);
	return(0);
}

con g++ mi restituisce, a parte alcuni warning, un errore del tipo:

/usr/bin/ld: Undefined symbols:
nnfw::Random::setSeed(long)
collect2: ld returned 1 exit status

VC++, invece, mi dice più o meno:

main.obj : error LNK2019: riferimento al simbolo esterno "public:  
static void__ccdec1 nnfw::Random::setSeed(long)" (? 
setSeed at Random@nnfw@@SAXJ at Z) non risolto nella funzione _main

Probabilmente è uno di quei problemi idioti che si risolvono in due  
secondi, ma sto googlando da ieri senza trovare una soluzione...
